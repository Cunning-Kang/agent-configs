# /agent-plan

Create a new agent execution plan with discovery and contracts.

## Usage

```
/agent-plan [objective] [--discovery] [--no-discovery]
```

## Arguments

- `objective`: The high-level task or goal to plan for (optional, can be provided in subsequent prompts)

## Options

- `--discovery`: Force codebase discovery before planning
- `--no-discovery`: Skip codebase discovery (use when repo context is well-known)

## Behavior

1. **Decide discovery need**: Determine if codebase discovery is required based on:
   - Complexity of the objective
   - Prior knowledge of the codebase
   - Whether relevant files/tests are already known

2. **Call codebase-discovery** (if needed):
   - Invoke the `codebase-discovery` subagent
   - Store output in `.agent-runs/plans/{plan_id}/discovery/codebase_discovery.json`
   - Discovery output is advisory only

3. **Create `plan.json`**:
   - Location: `.agent-runs/plans/{plan_id}/plan.json`
   - Contains plan metadata, contract references
   - Static manifest only — no runtime state

4. **Create initial contracts**:
   - Location: `.agent-runs/plans/{plan_id}/contracts/`
   - `contract-001.json` with status `proposed`
   - Opus main synthesizes contract content from discovery + objective

## Output

```
Created plan: plan-YYYYMMDD-NNN
├── plan.json
├── discovery/
│   └── codebase_discovery.json (if discovery was run)
└── contracts/
    └── contract-001.json
```

## Notes

- This command **never executes tasks** — it only creates the planning artifacts
- Contract status starts as `proposed` unless Opus main explicitly approves
- `/agent-run` only accepts contracts with status `approved`
- Task IDs are generated by `agent-loop init-run`, not during planning
